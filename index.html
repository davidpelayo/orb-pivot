Code Examples

From a simple HTML page to advanced queries and custom aggregation.

HTML 
<!-- 1. Styles -->
<link rel="stylesheet"
  href="node_modules/orb-pivot/dist/orb.css" />

<!-- 2. React 18 -->
<script src="unpkg.com/react@18/umd/react.min.js"></script>
<script src="unpkg.com/react-dom@18/umd/react-dom.min.js"></script>

<!-- 3. orb-pivot -->
<script src="node_modules/orb-pivot/dist/orb.min.js"></script>

<div id="pivot"></div>

<script>
  var widget = new orb.pgridwidget({
    dataSource: [
      ['US', 'Laptops', 120, 89999],
      ['US', 'Phones',  200, 159800],
      ['UK', 'Laptops',  90, 67499],
      ['UK', 'Phones',  180, 143820]
    ],
    fields: [
      { name: '0', caption: 'Country' },
      { name: '1', caption: 'Product' },
      { name: '2', caption: 'Qty' },
      { name: '3', caption: 'Revenue',
        dataSettings: { aggregateFunc: 'sum' } }
    ],
    rows:    ['Country'],
    columns: ['Product'],
    data:    ['Qty', 'Revenue']
  });
  widget.render(document.getElementById('pivot'));
</script>

How it works

  • Load orb.css for pivot styling
  • Include React 18 UMD builds (peer dependency)
  • Include the orb.min.js UMD bundle
  • Define data as a 2D array of rows
  • Map array indices to field captions
  • Assign fields to rows, columns, and data
  • Call render() on a DOM container

Config options

  • width — fixed width in pixels
  • grandTotal — row/column grand totals
  • subTotal — show/collapse sub totals
  • dataHeadersLocation — 'rows' or 'columns'
  • theme — built-in theme or Bootstrap
JavaScript 
import orb from 'orb-pivot';
import 'orb-pivot/dist/orb.css';

// Create a pivot grid (data model, no UI)
const pgrid = new orb.pgrid({
  dataSource: salesData,
  fields: [
    { name: '0', caption: 'Country' },
    { name: '1', caption: 'Product' },
    { name: '2', caption: 'Revenue',
      dataSettings: { aggregateFunc: 'sum' } }
  ],
  rows:    ['Country'],
  columns: ['Product'],
  data:    ['Revenue']
});

// Fluent query API
const q = pgrid.query();

// Grand total
q.val('Revenue');
// => { Revenue: 461119.98 }

// Filter by dimension
q.Country('US').val('Revenue');
// => { Revenue: 249799.99 }

// Multiple fields at once
q.Country('US').val('Revenue', 'Qty');
// => { Revenue: 249799.99, Qty: 310 }

// Statistical aggregation
q.stdev('Revenue');
q.avg('Revenue');

The Query API

Use orb.pgrid to create a data model without UI. Then .query() returns a fluent query object you can chain.

Available methods

  • .FieldName(value) — filter by dimension
  • .val('F1', 'F2') — aggregated values
  • .avg() .sum() — built-in
  • .stdev() .var() — statistics
  • .min() .max() .count()

Key point

The query API works without any DOM — use it in Node.js scripts, data pipelines, or server-side analytics.

JavaScript 
var config = {
  dataSource: salesData,
  fields: [
    { name: '0', caption: 'Region' },
    { name: '1', caption: 'Product' },
    {
      name: '2',
      caption: 'Revenue',
      dataSettings: {
        aggregateFunc: 'sum',
        // Custom format: currency
        formatFunc: function(val) {
          return '$' + Number(val)
            .toLocaleString('en-US', {
              minimumFractionDigits: 2
            });
        }
      }
    },
    {
      name: '3',
      caption: 'Count',
      dataSettings: {
        // Custom aggregation
        aggregateFunc: function(
          datafield, intersection, datasource
        ) {
          return intersection.length;
        }
      }
    }
  ],
  rows: ['Region'],  columns: ['Product'],
  data: ['Revenue', 'Count'],
  grandTotal: { rowsvisible: true },
  width: 800
};

Custom format functions

formatFunc receives the aggregated value and returns a display string. Use it for currency, percentages, or any formatting.

Custom aggregation

Replace built-in aggregation with any function. Your function receives:

  • datafield — field definition
  • intersection — matching row indices
  • datasource — full data array

Return any numeric value. Common uses: weighted averages, distinct counts, percentiles, custom business metrics.

Multiple data fields

List multiple fields in data to display them side by side. Each field can have its own aggregation and format function independently.

JavaScript 
const orb = require('orb-pivot');

// Query any array — no pivot grid needed
const data = [
  ['US', 'Laptops', 'Acme', 'Economy', 89999],
  ['US', 'Phones',  'Acme', 'Premium', 159800],
  ['UK', 'Laptops', 'Beta', 'Economy', 67499],
  ['UK', 'Phones',  'Beta', 'Premium', 143820]
];

const fields = {
  '2': { caption: 'Manufacturer' },
  '3': { caption: 'Class' },
  '4': {
    toAggregate: true,
    caption: 'Amount',
    aggregateFunc: 'sum'
  }
};

// Chain filters, get result
orb.query(data, fields)
  .Manufacturer('Acme')
  .Amount();
// => 249799

orb.query(data, fields)
  .Manufacturer('Acme')
  .Class('Economy')
  .Amount();
// => 89999

No grid required

orb.query() works directly on any 2D array without creating a pivot grid or rendering UI.

How it works

  • Pass your data array and a field definition object
  • Keys are column indices, values define captions and aggregation
  • Set toAggregate: true on measure fields
  • Chain dimension filters, then call the measure name

Use cases

  • Quick data exploration in Node.js scripts
  • Server-side analytics without rendering
  • Lightweight aggregation in data pipelines
  • Unit testing business logic
Programmatic

Query API in 3 Steps

Aggregate data without rendering. Perfect for backends, pipelines, and analytics.

1

Create

Initialize with your data and fields.

const orb = require('orb-pivot');
const pgrid = new orb.pgrid(config);
2

Filter

Chain dimension filters to narrow results.

const q = pgrid.query()
  .Country('US')
  .Product('Laptops');
3

Aggregate

Get results with any aggregation.

q.val('Revenue', 'Qty');
// { Revenue: 89999, Qty: 120 }

q.stdev('Revenue');
// { Revenue: 1377.58 }
